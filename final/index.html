Addendum: Technical Implementation Details

In this addendum, we delve deeper into the technical aspects of the Lights Out game, focusing on the implementation details and challenges encountered during development.

  1. Dynamic Game Board Generation:
    • Challenge: Generating a dynamic game board with toggleable lights.
    • Solution:
      • Utilized JavaScript to dynamically create the game board using DOM manipulation.
      • Implemented nested loops to generate a grid of squares, each representing a light.
      • Assigned event listeners to each square for toggling its state when clicked.
  2. Random Initialization of Lights:
    • Challenge: Randomly setting the initial state of lights while ensuring solvability.
    • Solution:
      • Implemented a function, shuffleBoard(), to randomly toggle lights at the start of the game.
      • Used the Math.random() function to determine whether each light should be initially on or off.
      • Ensured that the game remains solvable by validating the initial board configuration.
  3. Win Condition and User Feedback:
    • Challenge: Checking for win conditions and providing feedback to the user.
    • Solution:
      • Developed the checkForWin() function to verify if all lights are turned off.
      • Displayed a congratulatory message and a modal popup upon winning the game.
      • Implemented modal functionality to allow the user to play again or exit the game.
